# 2. Запуск контейнерів Docker

>**Контейнер** — це стандартна одиниця програмного забезпечення, яка упаковує код та всі його залежності, завдяки чому програма працює швидко та надійно при завантаженні у будь-яке обчислювальне середовище.
>
> Детальніше: https://docs.docker.com/get-started/docker-concepts/the-basics/what-is-a-container/ 

Команди для керування контейнерами можна знайти у файлі [ContainerCMD.md](/2_RunningContainers/ContainerCMD.md).

## Крок 1. Вибір образу (image)

Для запуску контейнера потрібен образ, в якому буде описаний зміст майбутнього контейнера. Образи можна створювати самостійно (буде розглянуто пізніше) або брати з [Docker HUB](https://hub.docker.com/).

Приклади образів з [Docker HUB](https://hub.docker.com/):
- **hello-world** — https://hub.docker.com/_/hello-world
- **python** — https://hub.docker.com/_/python
- **nginx** — https://hub.docker.com/_/nginx
- **ubuntu** — https://hub.docker.com/_/ubuntu

## Крок 2. Завантаження образу (опціонально)

Перед тим, як запускати образ, його можна завчасно завантажити.

```bash
docker pull IMAGE_NAME
```

Якщо не виконати цей крок — Docker автоматично завантажить образ при запуску.

## Крок 3. Запуск контейнера

Для запуску контейнера використовується команда `docker run`.

Приклад її використання:
```bash
docker run -d -p 8080:80 --name mysite nginx
```

Що відбувається:
1. Docker перевіряє наявність образу nginx
2. Якщо образу немає — завантажує його
3. Створює контейнер
4. Призначає ім’я mysite
5. Проброс порту 8080 → 80
6. Запускає контейнер у фоновому режимі

Якщо необхідно підключитися до терміналу контейнера, використайте атрибут `-it` та вкажіть оболонку, яка буде використана для роботи (зазвичай `bash` або `sh`). Наприклад:

```bash
docker run -it ubuntu bash
```

## Крок 4. Перевірка запущених контейнерів

Для того, щоб перевірити запущені контейнери, використовується команда `docker ps`. Якщо Вас цікавлять всі контейнери, включаючи вимкнені, додайте атрибут `-a`.

```bash
# Показати тільки запущені контейнери
docker ps

# Показати всі контейнери
docker ps -a
```
