# Команди для керування контейнерами

## Перегляд контейнерів

```bash
docker ps [OPTIONS]
```

Дозволяє переглянути перелік активних контейнерів на сервері.

| Аргумент | Опис |
|----------|------|
| `-a` | Показати всі контейнери, включаючи зупинені |
| `-q` | Показати тільки ID контейнерів |


Приклади використання

```bash
docker ps
docker ps -a
```
---

## Запуск контейнерів
```bash
docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
```

Створює та запускає контейнер з образу `IMAGE` з вказаними `OPTIONS` та виконавши при запуску команду `COMMAND`.

Образи контейнерів можна знайти на [Docker HUB](https://hub.docker.com/) або створити самостійно.

| Аргумент | Опис |
|----------|------|
| `-d` | Запуск у фоновому режимі (detached mode) |
| `-it` | Інтерактивний термінал |
| `-p 8080:80` | Проброс порту — ваша машина : контейнер |
| `-v $(pwd):/app` | Монтування тому — ваша папка : папка контейнера |
| `-w /app` | Встановлення робочої директорії всередині контейнера |
| `--name myapp` | Задати власне ім’я контейнера |
| `-e KEY=VALUE` | Встановлення змінної середовища |
| `--rm` | Автоматично видалити контейнер після зупинки |
| `--network` | Підключення до певної мережі |

Приклад використання
```bash
docker run -d -p 8080:80 --name mysite nginx
```
---

## Виконання команди у запущеному контейнері
```bash
docker exec [OPTIONS] CONTAINER COMMAND [ARG...]
```

Виконує команду `COMMAND` всередині вже запущеного контейнера `CONTAINER`.

| Аргумент | Опис |
|----------|------|
| `-d` | Запуск команди у фоновому режимі (detached mode) |
| `-it` | Інтерактивний термінал |
| `-w PATH` | Встановлення робочої директорії всередині контейнера |
| `-e KEY=VALUE` | Встановлення змінної середовища |

Приклад використання
```bash
docker exec -it mysite bash
```
---

## Запуск існуючого контейнера
```bash
docker start [OPTIONS] CONTAINER [CONTAINER...]
```

Запускає один `CONTAINER` або кілька `[CONTAINER...]` контейнерів, які були створені раніше та зупинені.

| Аргумент | Опис |
|----------|------|
| `-a` | Підключитися до STDOUT/STDERR контейнера |
| `-i` | Підключити в інтерактивному режимі |

Приклад використання
```bash
docker start mysite
```
---

## Перезапуск контейнера
```bash
docker restart [OPTIONS] CONTAINER [CONTAINER...]
```

Перезапускае один `CONTAINER` або кілька `[CONTAINER...]` контейнерів: спочатку зупиняє їх, потім знову запускає.

| Аргумент | Опис |
|----------|------|
| `-t SECONDS` | Час очікування перед зупинкою |

Приклад використання
```bash
docker restart mysite
```

---

## Зупинка контейнерів

```bash
docker stop [OPTIONS] CONTAINER [CONTAINER...]
```

Зупиняє контейнер `CONTAINER` та контейнери `[CONTAINER...]`, надіславши сигнал завершення процесу (SIGTERM) з подальшим примусовим завершенням (SIGKILL) після таймауту.

| Аргумент | Опис |
|----------|------|
| `-t SECONDS` | Час очікування перед примусовим завершенням (за замовчуванням ~10 секунд) |

Приклад використання
```bash
docker stop mysite -t 30
```

---

## Примусова зупинка контейнерів

```bash
docker kill [OPTIONS] CONTAINER [CONTAINER...]
```

Негайно зупиняє `CONTAINER` та контейнери `[CONTAINER...]`, надсилаючи сигнал примусового завершення (SIGKILL).

| Аргумент | Опис |
|----------|------|
| `-s SIGNAL` | Вказати інший сигнал (наприклад, SIGTERM, SIGINT) |

Приклад використання
```bash
docker kill mysite
```

---

## Видалення контейнера

```bash
docker rm [OPTIONS] CONTAINER [CONTAINER...]
```

Видаляє один або кілька контейнерів `CONTAINER`. Контейнер має бути зупинений перед видаленням.

| Аргумент | Опис |
|----------|------|
| `-f` | Примусово зупинити та видалити контейнер |
| `-v`	| Видалити пов'язані томи (volumes) |
| `-l`	| Видалити посилання (links), якщо є |

Приклад використання
```bash
docker rm mysite

# Видалити запущений контейнер
docker rm -f mysite

# Видалити всі зупинені контейнери
docker rm $(docker ps -aq) 
```

## Очищення системи Docker

```bash
docker system prune [OPTIONS]
```

Видаляє невикористані ресурси Docker для звільнення місця.

| Аргумент | Опис |
|----------|------|
|-a | Видалити всі невикористані образи (не лише dangling) |
|-f | Пропустити підтвердження |
|--volumes | Видалити невикористані томи |

Приклад використання
```bash
# Видаляються:
# - зупинені контейнери
# - невикористані мережі
# - dangling-образи
# - кеш збірки
docker system prune

# Максимально очищає систему Docker
docker system prune -a --volumes -f
```

## Отримання інформації про об'єкт Docker

```bash
docker inspect [OPTIONS] NAME|ID [NAME|ID...]
```

Повертає детальну інформацію у форматі JSON про контейнер `NAME|ID`, образ, мережу або том.

| Аргумент | Опис |
|----------|------|
| -f, --format | Вивести тільки потрібні дані за допомогою шаблону |
| -s | Показати інформацію про розмір (для контейнерів) |

Приклад використання
```bash
docker inspect mysite

# Вивести тільки IP-адресу контейнера
docker inspect -f "{{.NetworkSettings.IPAddress}}" mysite

```
